# USB·다른 컴퓨터에서 Cursor로 이어서 작업하기

이 문서는 **`배우 임승준 페이지` 폴더 전체**를 USB에 담아 다른 PC로 옮긴 뒤, **Cursor AI**로 같은 작업을 이어갈 때 필요한 내용을 정리한 것입니다.

---

## 1. USB에 반드시 넣을 것 (전체 폴더)

| 구분 | 설명 |
|------|------|
| **이 폴더 통째로** | `index.html`, `styles.css`, `assets/`, `README.md`, 배치 파일 등 **보이는 파일 전부** |
| **숨김 폴더·파일** | **`.git`** (Git 이력·브랜치·원격 저장소 주소), **`.github`** (Pages 배포 워크플로), **`.gitignore`**, **`.nojekyll`** |

### Windows에서 숨김까지 복사하려면

1. 탐색기에서 **보기** → **표시** → **숨긴 항목** 체크  
2. 또는 USB로 복사할 때 **상위 폴더 `COOT_Ai` 또는 `배우 임승준 페이지`를 통째로** 복사 (압축 ZIP으로 보내도 됨)

**`.git`이 빠지면** 이 컴퓨터에서 쌓아 둔 **커밋 기록·GitHub 연결(`origin`)**이 사라집니다. 이어서 버전 관리하려면 **반드시 포함**하세요.

---

## 2. 다른 PC에서 Cursor로 열기

1. USB의 **`배우 임승준 페이지`** 폴더를 PC로 복사 (로컬 디스크에 두는 것을 권장: USB 직접 열보다 빠름)  
2. **Cursor** 실행 → **File → Open Folder** (폴더 열기)  
3. 위 폴더를 선택  

프로젝트 루트가 이 폴더이면 `index.html`, `styles.css` 검색·편집·AI 채팅이 그대로 이어집니다.

---

## 3. Git / GitHub (이력·백업)

| 항목 | 설명 |
|------|------|
| **커밋 이력** | `.git` 폴더가 있으면 **이전 커밋·브랜치(`main` 등)** 유지 |
| **원격 저장소** | `git remote -v` 로 `https://github.com/imseungjun/imseungjun.git` 등이 보이면, 다른 PC에서 **push/pull** 가능 |
| **다른 PC에서 처음 push** | 해당 PC에 **Git** 설치 + **GitHub 로그인**(브라우저 또는 자격 증명 관리자) 필요할 수 있음 |
| **올리기 스크립트** | `깃허브에올리기.bat` — 원격 URL이 본인 저장소와 다르면 배치 파일 안의 주소를 수정 |

---

## 4. 브라우저에만 있는 데이터 (USB에 안 담김)

| 저장 위치 | 내용 |
|-----------|------|
| **`localStorage`** | 클립 링크·순서(`imseungjun_cliprows_v1`), 마이그레이션 플래그, 프로필 사진 편집 등 **브라우저마다 다름** |

즉, **다른 PC의 Chrome/Edge**를 열면 **그 PC에서 새로 입력한 것처럼** 비어 있거나 다를 수 있습니다.

**이어가려면 (선택):**

- 이전 PC 브라우저에서 **개발자 도구 → Application → Local Storage** 에서 값을 메모·스크린샷  
- 또는 중요한 링크만 `index.html` / 문서에 반영해 두기 (정적 파일은 USB와 Git에 함께 감)

---

## 5. 로컬 미리보기 (다른 PC)

- **`로컬미리보기.bat`** — 이 PC에 **Python**이 설치되어 있고 PATH에 잡혀 있어야 합니다.  
- 없으면: [python.org](https://www.python.org/) 에서 설치 후, 터미널에서 이 폴더로 이동해  
  `python -m http.server 8010`  
  → 브라우저에서 `http://127.0.0.1:8010/`

---

## 6. 공개 사이트 (GitHub Pages)

- 배포가 되어 있으면 예: **https://imseungjun.github.io/imseungjun/**  
- 설정 방법은 **`README.md`** 의 「인터넷 어디서나 링크로 보기」 참고.

---

## 7. 복사 전 체크리스트

- [ ] `배우 임승준 페이지` 안에 **`.git` 폴더**가 보이는가? (숨김 표시 후 확인)  
- [ ] `.github` 폴더가 있는가?  
- [ ] `assets` 폴더 용량이 큼 — USB 여유 공간 확인  
- [ ] 압축(ZIP)으로 보낼 경우, 압축 풀었을 때 **한 단계 위에 `index.html`이 바로 있게** 폴더 구조 유지  

---

## 8. 이 프로젝트에 없는 것 (참고)

- **Node / npm** — 없음. 설치 불필요.  
- **빌드 명령** — `index.html` 을 열면 됨 (단, YouTube 배경은 `file://` 보다 로컬 서버 권장).

---

요약: **숨김 포함해 폴더 전체 + Cursor에서 폴더 열기**만 하면 코드 작업은 그대로 이어갑니다. **브라우저에만 있던 클립 목록**은 별도로 챙겨야 다른 PC에서도 동일하게 보입니다.
